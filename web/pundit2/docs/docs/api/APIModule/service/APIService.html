APIService

  1. - service in module APIModule

The APIService service provides a global object to communicate with the application and some methods to add events and features to its.

The global object has on* methods which accept callbacks to get called in response to various events fired by the angular app through fireOn* methods. The app adds events by calling addEvent([names]) and the external users add their callbacks using GLOB.on*(function(){;}).

Example:

In app:

var api = APIService.init(confObject);
api.addEvent(['Open', 'Close']);
where appropriate: 
 api.fireOnOpen('argument', true)
and outside 
 GLOB.onOpen(function(arg1, arg2) {;})

The global object has call* methods which accept parameters to be passed to the angular app to run some internal feature/behavior.

The app is able to add such features using addFeature([names]) and exposing the code to run with expose*(function(){}).

The external users are able to call such exposed code with GLOB.call*().

Example:

In app

var api = APIService.init(confObject);
api.addFeature(['Open', 'Close']);
where appropriate 
 api.exposeOpen(function(arg) {  });
and outside 
 GLOB.callOpen('value')

Methods

  • addEvent(args);

    Adds on* (to service) and fireOn* (to global) methods.

    Parameters

    Param Type Details
    args arraystring

    String array or string, names of the events.

  • addFeature(args);

    Adds expose* (to service) and call* (to global) methods.

    Parameters

    Param Type Details
    args arraystring

    String array or string, names of the features.

  • init(conf);

    Initializes a new API service, exposing a global object on $window with a name taken from the configuration passed in.

    Configuration object.
